## 設備保養管理系統 - 系統需求文件

### 1. 系統總覽

本文件旨在闡述設備保養管理系統的系統規格與功能需求。此系統包含一個中央網頁管理平台以及一個供現場人員使用的行動應用程式(APP)，旨在數位化並優化設備巡檢、保養及維修的作業流程。系統將與現有的 AIMS 系統進行整合，以實現工單派送與資料回傳的自動化。

### 2. 系統架構

此系統採用主從式架構設計，由後端伺服器、網頁應用程式、行動應用程式及資料庫組成。

* **後端服務 (Server)**
    * **網站開發框架**: Flask 。
    * **伺服器資料庫**: PostgreSQL 16。
    * **部署環境**: GCP 。

* **前端應用 (Client)**
    * **網頁端**: 透過瀏覽器訪問的系統管理介面。
    * **行動端 (Mobile APP)**:
        * **Mobile APP 開發框架**: Flutter。
        * **本地資料庫**: APP 將使用 SQLite 進行資料本地儲存。
        * **運作模式**: 支援離線模式運作，允許在無網路環境下執行巡檢任務，並於網路恢復後同步資料 。

* **系統整合 (Integration)**
    * **AIMS 系統**: 透過 API 與 AIMS 系統介接 。
        * **資料查詢**: 包含工單查詢。
        * **資料回傳**: 將工單查詢結果與相關數據回傳至本系統 。

### 3. 功能性需求 - 網頁應用程式

網頁應用程式是系統的管理核心，提供全面的監控、查詢與管理功能。

* **3.1 使用者登入**
    * 提供使用者帳號與密碼的登入介面(預設帳密為admin / 1234qwer5T) 。
    * 系統名稱為「設備保養管理系統」 。
    * 支援 Azure Entra ID 帳號進行驗證 。

* **3.2 系統主頁 (Dashboard)**
    * 登入後顯示系統主頁，提供關鍵績效指標的快速預覽 。
    * **異常追蹤管理**: 顯示今日異常項目總數、累積異常未結案等統計數據。
    * **巡檢作業管理**: 顯示未完成、執行中、已完成的任務數量統計 。

* **3.3 巡檢作業管理**
    * **巡檢紀錄查詢**:
        * 使用者可依據開始日期與結束日期進行篩選查詢 。
        * 透過點選左側的組織樹狀結構進行資料搜尋 。
        * 查詢結果以列表顯示，包含詳細資料、異常、組織、巡檢路線、檢查日期、檢查人員等欄位 。
        * 可進一步點擊查看單筆紀錄的詳細資訊，如檢查設備、作業時間、到點時間、檢測人員等欄位資訊 。
        * 可鑽取至最底層，查看該次巡檢所有檢查項目的量測結果、上下限值、單位與檢查時間 。
    * **AIMS 工單執行進度查詢**:
        * AIMS為製造管理系統，本系統主要需要取得AIMS保養計劃以及巡檢計畫的工單資料，並派送給Mobile APP。
        * 提供多重篩選條件，如組織、工單日期、馬達類型、保養週期及工單號碼 。
        * 查詢結果列表呈現工單號碼、工單日期、組織、馬達類別等資訊 。
        * 可點擊特定工單，查看其中包含的設備代號、設備名稱、完成率、到點時間與人員 。
        * 可再深入查看單一設備的詳細檢查項目，包含下限值、上限值與檢查結果 。
        * 支援提供免驗證的對外公開檢視介面，允許 AIMS 透過特定 URL（帶有 `VI_NO` 參數）直接開啟獨立專屬頁面，查看單一工單之量測詳細報告。

* **3.4 組織與表單管理**
    * 系統需能以樹狀結構呈現公司的組織架構。
    * 每個組織單位底下可對應多張巡檢表單。

### 4. 功能性需求 - 行動應用程式 (Mobile APP)

行動應用程式（APP）專為現場人員設計，用於執行巡檢任務與資料收集。

* **4.1 登入與任務下載**
    * 提供 APP 登入畫面，名稱為「Smart 設備保養」。
    * 包含下載與設定功能按鈕 。
    * 登入後，主畫面會列出已分派的巡檢單，包含巡檢、定保、修護單等類型 。
    * 每張工單會以環圈圖顯示目前的完成度 。

* **4.2 巡檢執行與資料收集**
    * 點選工單後，進入設備選擇頁面。
    * 選擇設備後，顯示該設備的所有檢查項目，如前軸承溫度、馬達本體溫度等。
    * 提供狀態標記功能，如備機、檢修、停機。
    * **數據量測**:
        * 點選具體的檢查項目後，會跳出量測介面。
        * 介面會顯示量測項目的上下限值。
        * 透過「按壓採集」按鈕進行數據收集，暗示設備具備直接量測的能力。
    * **異常處理**:
        * 若無法量測或發生異常，使用者可選擇異常原因。
        * 支援拍照上傳功能作為佐證 。

### 5. 非功能性需求

* **5.1 基礎設施**
    * **資料庫**: 需建立獨立的測試區與正式區資料庫環境。例如，測試區 `PGDEVXXX,5432` ，正式區 `APPPGXXX,5432` 。
    * **資料庫驗證**: 使用本地帳號與密碼驗證 (Password Authentication)，遵循最小權限原則 。
    * **驅動程式**: 需安裝 `psycopg2-binary` 驅動程式 。

* **5.2 網路與安全**
    * **網域設定**: 需設定系統的網域名稱 (DNS) 。
    * **防火牆**: 需開啟網站服務 (Web) 與資料庫 (PostgreSQL) 所需的通訊埠 。
    * **資訊安全**: 系統所有網路傳輸需使用 SSL/TLS 憑證進行加密 。

* **5.3 維運管理**
    * 需建立標準化的部署流程、資料庫遷移策略與備份策略 。
    * 系統應具備日誌 (Logging) 記錄功能與完善的系統監控機制 。
