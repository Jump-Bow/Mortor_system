# 設備保養管理系統 後台文件庫導覽

為了保持文件脈絡清晰與避免混合放置，本 `docs/` 資料夾嚴格實施第一性原理與 MECE 歸類原則，所有知識庫文件均依據對象與生命週期分門別類：

## 目錄結構與規範

* 📂 **`architecture/` (核心架構與開發規範)**
  * **內容**：核心架構演進、MVC 模式、資料庫設計圖與開發環境設定。
  * **適用人員**：新進人員 (Onboarding)、架構團隊、所有開發者。
  * **性質**：長青文件，系統的基礎法則。

* 📂 **`api_and_data/` (介面協議與資料結構)**
  * **內容**：API 通訊規格、資料庫 Schema、資料模型與詳盡系統規格 (`SYSTEM_SPEC.md`)。
  * **適用人員**：前後端介接開發者、QA。
  * **性質**：經常需要參照的合約。

* 📂 **`hardware_integration/` (專業領域與硬體)**
  * **內容**：任何硬體整合的文件（此目錄保留為與 APP 專案文件對齊而建置）。
  * **性質**：高技術門檻的底層介接知識。

* 📂 **`features/` (模組功能規格)**
  * **內容**：依據業務邏輯劃分之功能說明 (如：系統管理規範、系統原始需求定義)。
  * **適用人員**：功能實作者、專案負責人 (PM)。
  * **性質**：功能與商業邏輯文件。

* 📂 **`logs_and_analysis/` (歷史紀錄與調查分析)**
  * **內容**：版本重大演進日誌 (Change Logs)、特定架構重構紀錄或調查報告。
  * **適用人員**：系統維護者。
  * **性質**：具強烈時效性，檔名必包含撰寫日期前綴 (如 `YYYYMMDD_*.md`)，並使用 YAML Frontmatter 標明文件類型 (type) 與適用對象 (audience)。

> ⚠️ 請遵照全域規範 `GEMINI.md` 進行維護，往後新增任何文件前務必三思其歸屬再進行歸檔。
